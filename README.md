## Task_5

Реализован тренировочный модуль **dev.site** для логирования изменений элементов инфоблоков в отдельный инфоблок LOG.  
Модуль подписан на события добавления и обновления элементов инфоблоков и формирует историю изменений с сохранением структуры разделов.

### Реализованный функционал
- Обработчик событий `OnAfterIBlockElementAdd` и `OnAfterIBlockElementUpdate`
- Автоматическое логирование изменений в инфоблок LOG
- Исключение логирования самого инфоблока LOG
- Формирование иерархии разделов (рекурсивно)
- Построение пути:
  **Инфоблок → Разделы (от родителя к ребёнку) → Элемент**
- Использование ID элемента как имени записи в LOG
- Запись даты создания/изменения в `ACTIVE_FROM`
- Автоматическое создание структуры разделов LOG при отсутствии
- Каждое изменение элемента создаёт новую запись в LOG (история изменений)

### Агент очистки
Реализован агент для автоматического обслуживания инфоблока LOG:
- Удаляет все записи, кроме 10 самых новых
- Запускается раз в 1 час
- Не удаляется из админки после установки
- Подключается через `InstallAgents()` модуля

### Используемые технологии
- CMS 1C-Битрикс
- PHP
- CIBlock API
- Bitrix EventManager
- Bitrix Agents
- Автозагрузка классов
- Работа с инфоблоками и разделами

### Структура модуля
- Модуль: `/local/modules/dev.site/`
- Установка модуля: `/local/modules/dev.site/install/index.php`
- Версия модуля: `/local/modules/dev.site/install/version.php`
- Автозагрузка: `/local/modules/dev.site/include.php`
- Обработчики событий: `/local/modules/dev.site/lib/Handlers/`
- Агент очистки: `/local/modules/dev.site/lib/Agents/`

### ⚙ Подключение
Модуль устанавливается через админку 1C-Битрикс. При установке автоматически:
- регистрируются обработчики событий
- создаётся агент очистки логов
- подключается автозагрузка классов
